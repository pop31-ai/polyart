Polynomial Lathe Sculpture: From Greek Amphorae to Game Assets
================================================================

Abstract
--------
Lathe sculpture - creating 3D forms by revolving 2D profiles - has
a rich history from Greek ceramic vessels to modern game assets.
This article develops polynomial representations for lathe profiles,
covering Greek vessel typology (amphora, kylix, krater), Da Vinci's
anatomical rotation studies, muscle relief mapping on surfaces of
revolution, and automated game asset generation pipelines. The
polyart.lathe module provides a complete toolkit for polynomial
lathe design, from historical vessel reconstruction to procedural
content generation.

1. Introduction
---------------
The potter's wheel, invented circa 3500 BCE, enabled the creation
of surfaces of revolution by spinning clay and applying a profile
tool. This process naturally generates polynomial curves: the tool
path is a continuous curve, and any smooth curve can be approximated
by polynomials. The same principle applies to CNC machining, 3D
printing, and digital sculpting. The polyart.lathe module captures
this principle mathematically, providing polynomial profile
generation and surface-of-revolution rendering.

2. Revolving Curve Profiles
-----------------------------
A surface of revolution is generated by rotating a planar curve
around an axis. Given a profile curve:

  C(t) = (r(t), z(t))

where r is the radius and z is the height, the surface of revolution
in 3D is:

  S(t, theta) = (r(t)*cos(theta), r(t)*sin(theta), z(t))

The surface normal (for rendering):

  N(t, theta) = normalize((-z'(t)*cos(theta), -z'(t)*sin(theta),
                           r'(t)))

The Gaussian curvature of a surface of revolution:

  K(t) = -z''(t) / (r(t) * (r'(t)^2 + z'(t)^2))

For polynomial profiles, all derivatives are themselves polynomials,
enabling exact curvature computation.

The polyart.lathe module generates surfaces of revolution:

  from polyart.lathe import LatheSurface

  profile = PolynomialProfile(r_coeffs=[1.0, 0.5, -0.3, 0.1],
                               z_range=(0, 10))
  surface = LatheSurface(profile, segments=64)
  surface.render('vessel.png')

3. Greek Vessel Typology
--------------------------
Ancient Greek pottery follows well-documented typological forms.
Each type has characteristic proportions expressible as polynomial
profiles.

3.1 Amphora
The amphora (storage vessel) has a profile with:
  - Narrow base (r ~ 0.3 * R_max)
  - Expanding lower body (belly)
  - Narrow neck (r ~ 0.25 * R_max)
  - Flared rim (r ~ 0.4 * R_max)
  - Two handles from neck to shoulder

Polynomial profile (degree 5):

  r(z) = a_0 + a_1*(z/H) + a_2*(z/H)^2 + a_3*(z/H)^3
       + a_4*(z/H)^4 + a_5*(z/H)^5

Characteristic coefficients for Attic amphora:
  a_0 = 0.30  (base radius)
  a_1 = 1.20  (body expansion)
  a_2 = -2.10 (body to shoulder transition)
  a_3 = 2.40  (shoulder to neck transition)
  a_4 = -1.80 (neck narrowing)
  a_5 = 0.60  (rim flare)

3.2 Kylix
The kylix (drinking cup) has a distinctive profile:
  - Wide, shallow bowl
  - Stem with node (knob)
  - Circular foot

The bowl profile uses a rational function approximation:

  r(z) = R_max * (z/H)^a / (1 + (z/H)^a)^(1/a)

where a ~ 2.5 gives the characteristic kylix curvature.

3.3 Krater
The krater (mixing vessel) combines amphora-like body with
wider mouth. Profiles for the three krater subtypes:

  Column krater: wide mouth, column handles, steep sides
  Bell krater: bell-shaped body, narrow base
  Volute krater: volute handles, wide mouth

Each subtype has a distinct polynomial signature in the
(r, z) profile plane.

3.4 Lekythos
The lekythos (oil flask) has:
  - Cylindrical body (r ~ constant)
  - Narrow neck
  - Single handle
  - Flat base

The profile is nearly a step function, approximated by:

  r(z) = R_cyl * (1 + tanh(k*(z - z_transition))) / 2

which approaches a constant above and below the transition.

4. Da Vinci Anatomical Studies
-------------------------------
Leonardo da Vinci's anatomical drawings include numerous studies
of body parts as surfaces of revolution. His rotating torso studies
treat the human trunk as a composition of lathe-like surfaces:

  - Head: near-spherical, r(z) ~ sqrt(R^2 - (z-z_c)^2)
  - Neck: cylindrical with polynomial taper
  - Thorax: ellipsoidal with ribs as lathe rings
  - Pelvis: truncated cone with polynomial smoothing
  - Limbs: tapered cylinders with muscle bulges

Da Vinci's note on proportion: "The profile of a column of the
human body is as follows: at the neck it is as thick as the face,
at the chest twice as thick, at the waist as thick as the neck."

This can be expressed as polynomial coefficients:
  neck_chest_ratio = 2.0
  neck_waist_ratio = 1.0

5. Muscle Relief Mapping
-------------------------
When applying anatomical detail to a lathe surface, muscle
profiles are mapped as radial displacements:

  r_displaced(z, theta) = r_base(z) + sum_i A_i * f_i(z, theta)

where f_i are muscle basis functions. For a lathe surface, the
theta-dependence is simplified by assuming approximate rotational
symmetry:

  f_i(z) = A_i * exp(-(z-z_peak_i)^2 / (2*sigma_i^2))

The polyart.lathe.muscle_relief() function applies muscle
displacements to a base profile:

  from polyart.lathe import LatheSurface, muscle_relief

  base = LatheSurface(cylinder_profile(r=1, h=3), segments=48)
  muscle_relief(base, muscles=[
      ('biceps', z_peak=1.5, A=0.15, sigma=0.3),
      ('triceps', z_peak=1.6, A=0.12, sigma=0.25),
      ('forearm', z_peak=0.8, A=0.10, sigma=0.2),
  ])

6. Game Asset Generation Pipeline
-----------------------------------
The pipeline: (1) Profile definition with vessel type/dimensions,
(2) Mesh generation with tessellation and UV mapping,
(3) Detail application with displacement and normal maps,
(4) Export to OBJ/FBX/glTF with LOD variants.

Usage:

  from polyart.lathe import GameAssetPipeline

  pipeline = GameAssetPipeline(output_format='glTF')
  pipeline.set_profile('amphora', height=0.5, radius=0.15)
  pipeline.add_handles(style='vertical', attachment='neck_shoulder')
  pipeline.tessellate(segments=32, rings=48)
  pipeline.bake_normal_map(high_poly_segments=128)
  pipeline.export('amphora_asset.glb')

7. Procedural Variation Generation
------------------------------------
The polynomial coefficients enable procedural variation:

  r_varied(z) = r_base(z) + epsilon * sum_{k} alpha_k * P_k(z)

where P_k are orthogonal polynomials (Legendre) and alpha_k are
random coefficients drawn from N(0, sigma_k^2). This generates
plausible vessel variations from a base type.

The variation space can be explored via:
  - Latent space interpolation between vessel types
  - Genetic algorithm optimization for target silhouettes
  - User-guided editing with real-time preview

8. Rendering Considerations
-----------------------------
Lathe surfaces have special rendering properties:

  - Perfect smoothness: analytical normals eliminate shading artifacts
  - Axis-aligned UV mapping: natural texture coordinates
  - Mirror symmetry: only one half needs tessellation
  - Open/closed ends: configurable cap generation

The polyart renderer handles these automatically, with options for:
  - PBR materials, subsurface scattering, environment mapping

9. Conclusion
--------------
Polynomial lathe profiles bridge ancient craft and modern digital
creation. From Greek amphorae to game assets, the surface of
revolution remains a fundamental geometric form. Polynomial
representation provides exact profiles, efficient tessellation,
and procedural variation capabilities. The polyart.lathe module
makes these capabilities accessible for historical reconstruction,
artistic creation, and real-time game content generation.
